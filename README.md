# 인터렉티브 키오스크와 메뉴 분석 추천 시스템 설계

본 프로젝트는 테이블마다 라즈베리 파이와 파이썬 GUI를 이용한 인터렉티브 키오스크(테이블오더)를 구현 및 Apriori 알고리즘을 이용한 장바구니 분석 기반 고객 맞춤 메뉴 추천 프로젝트에 대한 서술이다.

## 제작 동기

본 프로젝트를 시작된 계기는 필자가 일하는 매장에서 키오스크 도입을 고민 중이나, 높은 가격으로 인해(검색 시 대당 약 133만원, 월 사용료 약 2만원 지출로 추정, [출처](https://www.yna.co.kr/view/AKR20241225015800030)) 시스템을 구성하기 힘들다는 문제가 있었다. 해당 문제를 해결하기 위해서 미니 PC인 라즈베리 파이와 터치 가능한 화면이 달린 디스플레이 및 보조배터리 등을 포함, 테이블 당 가격을 20만원 선으로 낮추면서 [Google Colab](https://colab.google/)을 이용한 [Apriori](https://en.wikipedia.org/wiki/Apriori_algorithm) 분석으로 고객에게 추가 메뉴를 제안하는 시스템의 제작을 위해 시작하였다.

## 작동 방식

본 프로젝트의 경우 2 기계로 분리되어 구성된다. 두 기계는 포스기(메인 PC)를 Flask를 통해 서버로 이용, 매장에 있는 Wifi를 통해 통신을 하게 되기에, 와이파이 연결을 제외한 추가적인 네트워크 연결 과정은 필요 없다.

 - 포스기(메인 PC)
    - 메뉴 입력 & 데이터 전송용 GUI 포함 Python 파일(.exe 확장자로 배포)
    - 데이터 수집용 GUI 포함 Python 파일(.exe 확장자로 배포)
    - 데이터 분석용 Goggle Colab(Chrome 또는 Edge, 또는 파일 사용)

 - 라즈베리 파이(테이블오더용 PC)
    - 테이블오더 & 데이터 수집 & 데이터 배포용 Python 파일

### 포스기(메인 PC)

메인 PC의 경우 세 가지 역할을 수행한다.

1. 메뉴 입력 및 데이터 전송용 GUI 포함 Python 파일

해당 파일은 다음과 같은 역할을 한다.

 - 1. 새로운 메뉴 입력 및 삭제(JSON으로 관리).
 - 2. 추천 메뉴 분석 파일 입력(JSON).
 - 3. 원 버튼 클릭으로 라즈베리 파이의 메뉴 & 추천 메뉴 업데이트 및 (Flask를 이용한 Wifi 전송).

\[사진 추가 예정\]

매장을 관리하는 인원은 .exe 파일을 열어 GUI를 통해 메뉴를 관리 가능하다. 메뉴 입력의 경우 카테고리, 메뉴 이름, 사진, 가격, 설명 등을 입력한다. 해당 입력한 내용들은 JSON 파일로 저장되어 영업 개시시 연결된 모든 라즈베리 파이에게 전송된다.

3번에서 분석한 추천 메뉴 분석 결과를 JSON 파일 형태로 입력하여 연결된 모든 라즈베리 파이에게 전송한다.

영업 개시시 버튼을 클릭해 포스기와 동일한 Wifi에 연결된 모든 라즈베리 파이의 메뉴와 추천 메뉴(JSON)를 전송한다. 전송 과정의 경우 Flask를 통한 Wifi 전송을 사용하기에 라즈베리 파이에 .exe 파일이 켜져 있어야 한다.


2. 데이터 수집용 GUI 포함 Python 파일

해당 파일은 다음과 같은 역할을 한다.

 - 원 클릭으로 이용 내역(로그 수집)

해당 파일의 경우 1번과 비슷하게 버튼을 클릭해 포스기와 동일한 Wifi에 연결된 모든 라즈베리 파이의 데이터를 전송받는다.


3. 데이터 분석용 Goggle Colab

매장에 있는 포스기의 경우 성능이 좋지 않다는 것을 고려하여 클라우드 시스템인 Goggle Colab을 이용하여 분석을 진행한다.

2번에서 수집한 데이터를 넣어 알고리즘으로 처리해 결과 파일을 출력, 해당 파일을 1번 프로그램에 넣고 라즈베리 파이에 전송하는 역할을 하기 위한 분석 결과 생성을 한다.


### 라즈베리 파이(테이블오더용 PC)

테이블오더 특성상 디스플레이에서 GUI를 제공해야하기에 라즈베리 파이에서 Python을 이용한 GUI 구성 밎 데이터 수집이 이루어진다.

\[사진 추가 예정\]

Python의 CustomTkinter 라이브러리을 이용해 GUI 제공해 모던한 느낌의 UI를 고객에게 제공한다.  

매장 운영 중 구매한 데이터의 경우 내부 로그로 쌓이며, 메인 PC에서 데이터 수집 시 해당 로그들을 전송하게 된다.
